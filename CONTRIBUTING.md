Contributing
============

If you would like to contribute code to this project you can do so through GitHub by forking the repository and sending a pull request. Before RDK accepts your code into the project you must sign the RDK Contributor License Agreement (CLA).

## Getting Started

### Git Setup

If you're working with multiple GitHub accounts (e.g., personal and enterprise), or experiencing authentication issues when pushing changes, please refer to our [Git Setup and Credential Management Guide](GIT_SETUP.md).

The guide covers:
- Configuring Git for multiple accounts
- HTTPS and SSH authentication methods
- Troubleshooting common authentication errors (403 Forbidden, Permission denied)
- Best practices for credential management

### Contribution Workflow

1. **Fork the repository** on GitHub
2. **Clone your fork** to your local machine
3. **Configure Git** with your credentials (see [GIT_SETUP.md](GIT_SETUP.md))
4. **Create a feature branch** for your changes
5. **Make your changes** following the [Code Style Guidelines](CODE_STYLE.md)
6. **Test your changes** thoroughly
7. **Commit your changes** with clear, descriptive commit messages
8. **Push to your fork** on GitHub
9. **Create a pull request** to the main repository
10. **Sign the CLA** when prompted (required for first-time contributors)

### Code Quality

- Follow the coding standards outlined in [CODE_STYLE.md](CODE_STYLE.md)
- Ensure your code compiles without warnings
- Test your changes thoroughly before submitting
- Write clear commit messages that describe what and why, not just how
